Técnicas de programação e algorítmos

Técnicas de programação e algoritmos referem-se às metodologias e aos passos lógicos para resolver problemas de forma estruturada e eficiente através de um programa de computador. Um algoritmo é a "receita" ou a sequência de instruções para um objetivo, enquanto as técnicas de programação englobam as boas práticas e os recursos como variáveis, estruturas de controlo (condições e laços de repetição) e modularização (funções e procedimentos), que tornam o código claro e fácil de executar.


O que são algoritmos?

Um algoritmo é uma sequência finita de passos claros e bem definidos para resolver um problema específico ou atingir um objetivo.

Podem ser descritos em linguagem natural, pseudocódigo ou através de fluxogramas.


Tipos de algoritmos:

existem algoritmos sequenciais (passos em ordem), condicionais (tomam decisões) e de repetição (executam tarefas múltiplas vezes).


O que são técnicas de programação?

São os princípios e os métodos utilizados para construir programas de forma organizada e eficiente.


Recursos comuns:

Variáveis e tipos de dados: Para armazenar e manipular informações.


Estruturas de controlo:

Condicionais (If/Else):
Permitem ao programa tomar decisões com base em condições específicas.

Loops (Laços de repetição):
Permitem que o código execute uma ação repetidamente.

Modularização:
Organização de um programa em blocos menores (funções e procedimentos) para melhorar a legibilidade e reutilização do código.

Recursividade:
Uma função que chama a si mesma para resolver um problema.

Relação entre algoritmos e técnicas de programação

Os algoritmos fornecem o plano lógico para resolver um problema, e as técnicas de programação são as ferramentas e os métodos que se usam para transformar esse plano num programa funcional.

A lógica de programação é a base para criar algoritmos eficientes e para aplicar corretamente as técnicas de programação.


Exemplos de representação de algoritmos

Linguagem natural:
Uma descrição simples, em português comum, como: "1. Abra a porta. 2. Dê um passo para dentro".

Fluxograma:
Uma representação gráfica com símbolos e setas que mostram a sequência dos passos, incluindo decisões.

Pseudocódigo:
Uma linguagem intermediária, com uma estrutura semelhante a de uma programação, mas escrita de forma mais simplificada.
